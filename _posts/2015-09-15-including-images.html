Specifying the source

Positioning

Since HTML5, there are <figure> and <figcaption> tags.

Table-layouts, now called "grid-systems", have won. Fifteen years ago, we had tables everywhere, now we have div's with weird CSS-classes all over the place. CSS seems to be only slightly less insane than it was 10 years ago, positioning anything on the screen in a predictable way still seems nearly impossible to me.

Just in case that I should ever try to rewrite the style for the figure positioning, here is a list of strange rules to keep in mind:

  • You can, theoretically, center stuff with margin: auto;
  • It does not work if the width is not set explicitly
  • It does not work if the element has display not equal to block
Here is what I've come up with for now:
  1. For each row of figures, create a <div> with class row
  2. For each figure in the row, create a <figure> element with class col-xs-{12/n}, where n is the number of figures in the row.
  3. In each <figure>, insert the <img> and <figcaption> tags.
  4. Add class="framed" to both image and caption.
  5. Add style="max-width: <N>px;" to both image and caption. Keep this size consistent, otherwise the little rounded rectangle around the image will fall apart.
The framed class is defined in custom.css as follows: {% highlight css %} .framed { border-color: #BBBBBB; border-style: solid; background-color: #EEEEFF; padding: 5px; width: 100%; margin-left: auto; margin-right: auto; display: block; } img.framed { border-radius: 5px 5px 0px 0px; border-width: 1px 1px 0px 1px; } figcaption.framed { border-radius: 0px 0px 5px 5px; border-width: 0px 1px 1px 1px; } {% endhighlight %} It makes sure that both image and caption are centered within the figure, and that there is a rectangle with rounded border that groups the image and the caption together.
